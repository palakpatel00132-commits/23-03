Step 2: React ના પાયાના કોન્સેપ્ટ્સ ગુજરાતીમાં!

1. Functional Components 🏗️

React માં કમ્પોનન્ટ એટલે એક સાદું JavaScript ફંક્શન જે HTML (JSX) રિટર્ન કરે છે. આજના સમયમાં ક્લાસ કમ્પોનન્ટની જગ્યાએ આ જ વપરાય છે. 
function Welcome() {
    return (
        <div>
            <h1>Hello React!</h1>
            <p>આ એક ફંક્શનલ કમ્પોનન્ટ છે.</p>
        </div>
    );
}

2. useState Hook 🔢

જ્યારે કમ્પોનન્ટમાં કોઈ ડેટા બદલાય અને આપણે પેજને અપડેટ કરવું હોય, ત્યારે useState વપરાય છે.
રીયલ લાઈફ: સ્કોરબોર્ડ. રન વધે એટલે સ્કોરબોર્ડ પર આંકડો બદલાય, તેમ સ્ટેટ બદલાય એટલે UI બદલાય. 
import { useState } from 'react';

function Counter() {
    const [count, setCount] = useState(0);

    return (
        <div>
            <p>તમે {count} વાર ક્લિક કર્યું.</p>
            <button onClick={() => setCount(count + 1)}>વધારો</button>
        </div>
    );
}

3. useEffect Hook (VERY IMPORTANT ⚠️)

જ્યારે કમ્પોનન્ટ લોડ થાય, અપડેટ થાય કે બંધ થાય ત્યારે કોઈ કામ કરવું હોય (જેમ કે API કોલ), ત્યારે useEffect વપરાય છે. આમાં Dependency Array ના ૩ મુખ્ય કિસ્સાઓ (scenarios) છે:

૧. Dependency Array વગર (No Array):

જો તમે બીજો આર્ગ્યુમેન્ટ ના આપો, તો આ ઇફેક્ટ દરેક રેન્ડર (render) વખતે ચાલશે. 
useEffect(() => {
    console.log("હું દરેક વખતે ચાલીશ! 🔄");
});

૨. ખાલી Dependency Array (Empty Array []):

આ ઇફેક્ટ ફક્ત પહેલી જ વાર (on mount) ચાલશે. API કોલ કરવા માટે આ સૌથી વધુ વપરાય છે. 
useEffect(() => {
    console.log("હું ફક્ત પહેલી જ વાર ચાલીશ! 🚀");
}, []);

૩. વેરીએબલ સાથે (With Variables [data]):

જ્યારે એરેમાં લખેલો વેરીએબલ બદલાશે, ત્યારે જ આ ઇફેક્ટ ફરીથી ચાલશે. 
const [count, setCount] = useState(0);

useEffect(() => {
    console.log("જ્યારે count બદલાશે ત્યારે જ હું ચાલીશ! 📈");
}, [count]);
રીયલ લાઈફ: - No Array: શ્વાસ લેવો (દરેક પળે ચાલુ). - Empty Array: જન્મ લેવો (જીવનમાં ફક્ત એક જ વાર). - With Variables: ભૂખ લાગે ત્યારે જમવું (જ્યારે ભૂખ લાગે ત્યારે જ આ કામ થાય).

4. Props vs State 🔄

બંને ડેટા સાચવે છે પણ બંનેનો હેતુ અલગ છે:
  • Props: બહારથી (Parent માંથી) મળતો ડેટા. તે Read-only છે.
  • State: કમ્પોનન્ટનો પોતાનો Private ડેટા જે બદલી શકાય છે.
રીયલ લાઈફ: - Props: પપ્પાએ આપેલી ગિફ્ટ (તમે તેને બદલી ના શકો). - State: તમારા ખિસ્સામાં રહેલા પૈસા (તમે ગમે ત્યારે વાપરી કે વધારી શકો).

5. Child to Parent Communication ⬆️

React માં ડેટા હંમેશા ઉપરથી નીચે (Parent થી Child) જાય છે. જો Child માંમાંથી Parent ને ડેટા મોકલવો હોય, તો Parent એક ફંક્શન (Function) Child ને Props તરીકે આપે છે, અને Child તે ફંક્શનને કોલ કરીને ડેટા પાછો મોકલે છે.
રીયલ લાઈફ: વોકી-ટોકી. Parent (મેઈન સ્ટેશન) Child ને કહે છે "જ્યારે કામ પતી જાય ત્યારે મને આ બટન દબાવીને જણાવજો". Child બટન દબાવે (Function call કરે) એટલે Parent ને ખબર પડી જાય.

પ્રેક્ટિસ કોડ:

// Parent Component
function Parent() {
    const handleDataFromChild = (data) => {
        console.log("Child એ મોકલેલ ડેટા:", data);
    };

    return <Child onSend={handleDataFromChild} />;
}

// Child Component
function Child({ onSend }) {
    return (
        <button onClick={() => onSend("હેલો પપ્પા! 👋")}>
            Parent ને ડેટા મોકલો
        </button>
    );
}

6. Conditional Rendering 🚦

શરત (Condition) મુજબ અલગ-અલગ HTML બતાવવાની રીત. 
function Greeting({ isLoggedIn }) {
    return (
        <div>
            {isLoggedIn ? (
                <h1>વેલકમ બેક! 👋</h1>
            ) : (
                <h1>કૃપા કરીને લોગિન કરો. 🔑</h1>
            )}
        </div>
    );
}

7. Lists & Keys 🔑

જ્યારે તમારે એરે (Array) માંથી ડેટા લઈને લિસ્ટ બતાવવું હોય, ત્યારે map() ફંક્શન વપરાય છે. દરેક આઈટમને એક યુનિક Key આપવી જરૂરી છે જેથી React જાણી શકે કે કઈ આઈટમ બદલાઈ છે.
રીયલ લાઈફ: રોલ નંબર. ક્લાસમાં બધાના નામ સરખા હોઈ શકે, પણ રોલ નંબર અલગ હોય એટલે ટીચરને ખબર પડે કે કોણ કોણ હાજર છે. 
const fruits = ["🍎", "🍌", "🍇"];

return (
    <ul>
        {fruits.map((fruit, index) => (
            <li key={index}>{fruit}</li>
        ))}
    </ul>
);

8. Fragments (<> </>) 📦

React માં કમ્પોનન્ટ હંમેશા એક જ પેરેન્ટ એલિમેન્ટ રિટર્ન કરી શકે. જો તમારે વગર કારણે વધારાનો <div> ના ઉમેરવો હોય, તો તમે Fragment વાપરી શકો છો. 
return (
    <>
        <h1>હેલો</h1>
        <p>વધારાના div વગર!</p>
    </>
);
