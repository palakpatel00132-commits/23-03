Node.js Basics 🟢

Step 6: Node.js ના પાયાના કોન્સેપ્ટ્સ ગુજરાતીમાં!

1. Node.js Basics 🟢

Node.js એ JavaScript રનટાઈમ છે જે બ્રાઉઝરની બહાર (સર્વર પર) JS કોડ ચલાવવા માટે વપરાય છે. તે Chrome ના V8 એન્જિન પર બનેલું છે.
રીયલ લાઈફ: JavaScript પહેલા માછલી જેવી હતી જે ફક્ત પાણી (બ્રાઉઝર) માં જ રહી શકતી હતી. Node.js એ માછલીને જમીન (સર્વર) પર ચાલવા માટેનું 'હેલ્મેટ' અથવા સાધન આપ્યું!

મુખ્ય મુદ્દાઓ:

  • Single-threaded: એક જ સમયે એક જ કામ કરે.
  • Asynchronous: વાર લાગે તેવા કામો માટે અટકતું નથી.
  • Fast & Scalable: સર્વર સાઈડ એપ્લિકેશન માટે બેસ્ટ.

2. Event Loop (ઇવેન્ટ લૂપ) 🔄

Node.js સિંગલ થ્રેડેડ હોવા છતાં કેવી રીતે આટલું ઝડપી છે? તેનું રહસ્ય Event Loop છે. તે સતત ચેક કરે છે કે કોઈ કામ પત્યું છે કે નહીં, અને પત્યું હોય તો તેનું રિઝલ્ટ આપે છે.
રીયલ લાઈફ: એક હોટલનો વેઈટર. તે ઓર્ડર લઈને રસોડામાં આપી દે છે અને બીજા ટેબલ પર જાય છે. જ્યારે રસોડામાં જમવાનું તૈયાર થાય, ત્યારે તે પાછો જઈને પીરસી દે છે. તે રસોડામાં ઉભો નથી રહેતો! 
console.log("શરૂઆત");

setTimeout(() => {
    console.log("હું થોડી વાર પછી આવીશ (Event Loop)");
}, 0);

console.log("અંત");

// Output:
// શરૂઆત
// અંત
// હું થોડી વાર પછી આવીશ (Event Loop)

3. Non-blocking I/O ⚡

Node.js માં જ્યારે કોઈ ફાઇલ વાંચવાની હોય કે API કોલ કરવાનો હોય, ત્યારે તે કામ પતે ત્યાં સુધી પ્રોગ્રામ અટકતો નથી. તે આગળના કામ કરવા માંડે છે. 
const fs = require('fs');

// Non-blocking (Asynchronous)
fs.readFile('data.txt', 'utf8', (err, data) => {
    if (err) throw err;
    console.log(data);
});

console.log("ફાઇલ વંચાય છે, પણ હું અટક્યો નહીં! ✅");

4. Modules (CommonJS) 📦

Node.js માં કોડને અલગ-અલગ ફાઇલોમાં વહેંચવા માટે Modules વપરાય છે. જૂની અને પ્રચલિત રીત CommonJS છે જેમાં require અને module.exports વપરાય છે.

પ્રેક્ટિસ કોડ:

// math.js (Module બનાવવું)
const add = (a, b) => a + b;
module.exports = add;

// app.js (Module વાપરવું)
const add = require('./math');
console.log(add(5, 10)); // 15

5. setTimeout vs setInterval (Timers) ⏰

જાવાસ્ક્રિપ્ટમાં સમય સાથે કામ કરવા માટે બે મુખ્ય ટાઈમર છે. બંનેનો ફર્ક સમજવો બહુ જરૂરી છે:

૧. setTimeout (એક જ વાર):

આ ફંક્શન આપેલા સમય પછી ફક્ત એક જ વાર ચાલે છે.
એનાલોજી: એલાર્મ (Alarm). સવારે ૭ વાગ્યાનું એલાર્મ વાગે એટલે કામ પૂરું, તે ફરીથી વાગતું નથી. 
setTimeout(() => {
    console.log("હું ૨ સેકન્ડ પછી ફક્ત એક જ વાર દેખાઈશ! 👋");
}, 2000);

૨. setInterval (વારંવાર):

આ ફંક્શન આપેલા સમયના અંતરે વારંવાર (repeatedly) ચાલ્યા જ કરે છે, જ્યાં સુધી તમે તેને રોકો નહીં.
એનાલોજી: ઘડિયાળના કાંટાનું ટીક-ટીક. તે દર ૧ સેકન્ડે થયા જ કરે છે. 
const timer = setInterval(() => {
    console.log("હું દર ૨ સેકન્ડે દેખાયા કરીશ! 🔄");
}, 2000);

// રોકવા માટે:
// clearInterval(timer);

6. Express.js Basics 🚀

Express એ Node.js માટેની એક વેબ ફ્રેમવર્ક છે જે સર્વર બનાવવાનું કામ એકદમ સરળ કરી દે છે. Node.js ના ઇન-બિલ્ટ http મોડ્યુલ કરતા આમાં કોડ લખવો ઘણો સહેલો છે.
એનાલોજી: Node.js એટલે કાચો માલ (ઈંટો, સિમેન્ટ), જ્યારે Express એટલે તૈયાર બંગલો અથવા ફ્લેટ જેમાં બધી સુવિધાઓ પહેલેથી જ છે! 
const express = require('express');
const app = express();

app.listen(3000, () => {
    console.log("સર્વર ચાલુ થઈ ગયું! 🚀");
});

7. Routing (રાઉટિંગ) 🛣️

યુઝર કયા URL પર જાય છે તે મુજબ તેને શું બતાવવું તેને રાઉટિંગ કહેવાય.
  • app.get('/'): હોમ પેજ માટે.
  • app.post('/login'): ડેટા સબમિટ કરવા માટે.
app.get('/', (req, res) => {
    res.send("હેલો, આ હોમ પેજ છે! 👋");
});

app.get('/about', (req, res) => {
    res.send("આ અમારા વિશેની માહિતી છે. ℹ️");
});

8. Middleware (મિડલવેર) 🛡️

મિડલવેર એવા ફંક્શન છે જે રિક્વેસ્ટ (Request) સર્વર સુધી પહોંચે તે પહેલા વચ્ચે ચાલે છે. તે ડેટા ચેક કરવા કે લોગ કરવા માટે વપરાય છે.
એનાલોજી: એક સિક્યુરિટી ગાર્ડ. તમે કોઈ ઓફિસમાં જાઓ તે પહેલા ગાર્ડ તમારું નામ-પત્તું ચેક કરે છે, તે મિડલવેર છે! 
app.use((req, res, next) => {
    console.log("નવી રિક્વેસ્ટ આવી! 📨");
    next(); // આગળ વધવા માટે આ જરૂરી છે!
});

9. Request & Response 📨

- req (Request): જે ડેટા ક્લાયન્ટ (બ્રાઉઝર) મોકલે છે. (URL, Body, Headers).
- res (Response): જે ડેટા સર્વર પાછો મોકલે છે. (JSON, HTML, Status Code). 
app.get('/user/:id', (req, res) => {
    const id = req.params.id; // URL માંથી ID લેવો
    res.status(200).json({ userId: id, name: "Palak" });
});

10. Error Handling ⚠️

જ્યારે સર્વરમાં કોઈ ભૂલ આવે, ત્યારે યુઝરને મસ્ત મેસેજ બતાવવા માટે એરર હેન્ડલિંગ વપરાય છે. 
// ખાસ એરર હેન્ડલિંગ મિડલવેર (૪ આર્ગ્યુમેન્ટ્સ)
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send("કંઈક ભૂલ થઈ છે! ❌");
});

11. REST API Concepts 🌐

REST (Representational State Transfer) એ API બનાવવાની એક આર્કિટેક્ચર સ્ટાઇલ છે. તે સ્ટેટલેસ (Stateless) હોય છે, એટલે કે દરેક રિક્વેસ્ટમાં બધી જ માહિતી હોવી જોઈએ.

૧. CRUD Operations:

API દ્વારા ડેટા સાથે ૪ મુખ્ય કામ કરી શકાય છે:

  • Create (POST): નવો ડેટા બનાવવા માટે.
  • Read (GET): ડેટા મેળવવા માટે.
  • Update (PUT/PATCH): જૂનો ડેટા બદલવા માટે.
  • Delete (DELETE): ડેટા કાઢી નાખવા માટે.

૨. Common Status Codes:

સર્વર રિક્વેસ્ટનો જવાબ એક કોડ દ્વારા આપે છે:

  • 200 OK: બધું બરાબર પત્યું.
  • 201 Created: નવો ડેટા સફળતાપૂર્વક બની ગયો.
  • 400 Bad Request: તમારી રિક્વેસ્ટમાં કંઈક ભૂલ છે.
  • 401 Unauthorized: તમારે લોગિન કરવું પડશે.
  • 404 Not Found: આ પેજ કે ડેટા મળ્યો નથી.
  • 500 Server Error: સર્વરમાં કંઈક લોચો થયો છે.

૩. API Structure:

એક સારી API હંમેશા સાફ અને ક્લિયર હોવી જોઈએ. 
// સાચું Structure:
// GET    /api/users          -> બધા યુઝર્સ મેળવો
// GET    /api/users/123      -> ID 123 વાળો યુઝર મેળવો
// POST   /api/users          -> નવો યુઝર બનાવો
// PUT    /api/users/123      -> ID 123 વાળો યુઝર અપડેટ કરો
// DELETE /api/users/123      -> ID 123 વાળો યુઝર ડિલીટ કરો
એનાલોજી: એક પુસ્તકાલય (Library). - GET: ચોપડી વાંચવી. - POST: નવી ચોપડી લાવીને મૂકવી. - PUT: ફાટેલી ચોપડી બદલીને નવી મૂકવી. - DELETE: જૂની ચોપડી કાઢી નાખવી. - Status Code: લાઈબ્રેરિયનનો જવાબ (ચોપડી મળી, નથી મળી, કે તમે મેમ્બર નથી).

12. MongoDB & NoSQL Basics 🗄️

૧. SQL vs NoSQL (તફાવત):

ફીચર SQL (MySQL, Postgre) NoSQL (MongoDB)
Structure Table (Row/Column) Document (JSON/BSON)
Schema Fixed (પહેલેથી નક્કી) Flexible (ગમે ત્યારે બદલાય)
Scaling Vertical (મોટું કમ્પ્યુટર) Horizontal (વધારે કમ્પ્યુટર)

૨. MongoDB Structure:

  • Database: આખું કન્ટેનર.
  • Collection: ટેબલ જેવું (દા.ત. Users).
  • Document: એક ડેટા એન્ટ્રી (JSON જેવી).

૩. CRUD Operations:

// Create
db.users.insertOne({ name: "Palak", age: 25 });

// Read
db.users.find({ name: "Palak" });

// Update
db.users.updateOne({ name: "Palak" }, { $set: { age: 26 } });

// Delete
db.users.deleteOne({ name: "Palak" });

૪. Indexing (ઇન્ડેક્સિંગ):

જ્યારે લાખો ડેટા હોય, ત્યારે સર્ચિંગ ફાસ્ટ કરવા માટે Indexing વપરાય છે. 
db.users.createIndex({ name: 1 }); // નામ પર ઇન્ડેક્સ બનાવવો
એનાલોજી: - SQL: એક રજીસ્ટર જેમાં લીટીઓ અને ખાના પાડેલા છે. - NoSQL (MongoDB): એક કબાટ (Database) જેમાં ફાઈલો (Collection) છે, અને દરેક ફાઈલની અંદર અલગ-અલગ કાગળો (Document) છે. કાગળમાં તમે ગમે તેટલી માહિતી લખી શકો! - Indexing: ચોપડીના છેલ્લે આપેલું 'Index' પેજ, જેથી તમારે આખી ચોપડી વાંચવી ના પડે અને સીધું પેજ મળી જાય.

13. Mongoose (Object Data Modeling) 🔗

Mongoose એ MongoDB માટેની એક લાઈબ્રેરી છે જે ડેટાને સ્ટ્રક્ચર (Schema) આપવામાં અને વેલિડેશન કરવામાં મદદ કરે છે.

૧. Schema & Model:

  • Schema: ડેટાનું માળખું (Blueprint) કે ડેટા કેવો દેખાશે.
  • Model: તે સ્કીમા પરથી બનેલું ઓબ્જેક્ટ જેનાથી આપણે ડેટાબેઝ સાથે વાત કરીએ છીએ.
const mongoose = require('mongoose');

const userSchema = new mongoose.Schema({
    name: { type: String, required: true },
    age: Number
});

const User = mongoose.model('User', userSchema);

૨. Validation (વેલિડેશન):

ડેટાબેઝમાં ખોટો ડેટા ના જાય તે ચેક કરવાની રીત. 
const userSchema = new mongoose.Schema({
    email: {
        type: String,
        required: [true, "Email જરૂરી છે!"],
        unique: true,
        lowercase: true
    },
    password: { type: String, minlength: 6 }
});

// વાપરતી વખતે:
const post = await Post.find().populate('author'); // આનાથી 'author' ની બધી વિગતો મળી જશે!

૩. Relationships (સંબંધો):

બે અલગ-અલગ ડેટા (Collections) ને જોડવા માટે ref અને populate વપરાય છે.
  • SQL: આમાં Foreign Key અને Join વપરાય છે.
  • Mongoose: આમાં ref (ID સાચવવા) અને populate (બીજા ટેબલનો ડેટા ખેંચવા) વપરાય છે.
// Post Schema (User સાથે જોડવું)
const postSchema = new mongoose.Schema({
    title: String,
    author: { type: mongoose.Schema.Types.ObjectId, ref: 'User' }
});

// વાપરતી વખતે:
const post = await Post.find().populate('author'); // આનાથી 'author' ની બધી વિગતો મળી જશે!
એનાલોજી: - Schema: મકાનનો નકશો (કયો રૂમ ક્યાં હશે તે પહેલેથી નક્કી). - Model: તૈયાર થયેલું મકાન જેમાં તમે રહી શકો. - Populate: એક ફાઈલમાં બીજી ફાઈલનું એડ્રેસ લખેલું હોય, અને તમે તે એડ્રેસ પર જઈને બીજી ફાઈલની માહિતી પણ લઈ આવો.

14. Authentication (ઓથેન્ટિકેશન) 🔐

ઓથેન્ટિકેશન એટલે યુઝર કોણ છે તેની ખાતરી કરવી. આમાં બે મુખ્ય વસ્તુઓ છે: પાસવર્ડને સુરક્ષિત રાખવો અને લોગિન સેશન સાચવવું.

૧. Password Hashing (bcrypt):

ક્યારેય પાસવર્ડને ડેટાબેઝમાં સાદો (Plain text) ના રખાય. bcrypt લાઈબ્રેરી પાસવર્ડને અઘરા કોડમાં ફેરવી નાખે છે. 
const bcrypt = require('bcrypt');

// પાસવર્ડ સેવ કરતી વખતે (Hash)
const hashedPassword = await bcrypt.hash("myPassword123", 10);

// લોગિન વખતે (Compare)
const isMatch = await bcrypt.compare("myPassword123", hashedPassword);

૨. JWT (JSON Web Token):

જ્યારે યુઝર લોગિન થાય, ત્યારે સર્વર એક Token આપે છે. આ ટોકન બ્રાઉઝર સાચવી લે છે અને દરેક રિક્વેસ્ટ સાથે મોકલે છે. 
const jwt = require('jsonwebtoken');

// ટોકન બનાવવો (Sign)
const token = jwt.sign({ userId: user._id }, 'secret_key', { expiresIn: '1h' });

// ટોકન ચેક કરવો (Verify)
const decoded = jwt.verify(token, 'secret_key');

૩. Login / Signup Flow:

Signup: User Data ➔ Hash Password ➔ Save to DB.
Login: Check User ➔ Compare Hash ➔ Generate JWT Token ➔ Send to User.
એનાલોજી: - Password Hashing: તમારો પાસવર્ડ એક 'મિક્સર' માં નાખીને તેનો જ્યુસ બનાવી દેવો. જ્યુસ પરથી પાછો આખો ફ્રૂટ (પાસવર્ડ) કોઈ ના બનાવી શકે! - JWT: મુવીની ટિકિટ. તમે પૈસા આપીને ટિકિટ લો (Login), પછી આખું મુવી જોતી વખતે વારંવાર પૈસા ના આપવા પડે, ફક્ત ટિકિટ બતાવો એટલે ચાલે.

15. API Integration (Backend to Backend) 📦

જ્યારે તમારું સર્વર (Node.js) બીજા કોઈ સર્વર (API) સાથે વાત કરે છે, ત્યારે તેને Backend-to-Backend API Integration કહેવાય છે.

૧. Fetch / Axios (Backend side):

- Native Fetch: Node.js 18+ માં પહેલેથી જ આવે છે.
- Axios: એક્સ્ટ્રા લાઈબ્રેરી છે જે એરર હેન્ડલિંગ અને સિક્યુરિટીમાં વધુ સારી છે. 
const axios = require('axios');

const getExternalData = async () => {
    try {
        const response = await axios.get('https://api.example.com/data');
        console.log(response.data);
    } catch (error) {
        console.error("ભૂલ આવી: ", error.message);
    }
};

૨. Handling API Errors:

બીજા સર્વર પરથી ડેટા મંગાવતી વખતે હંમેશા try...catch બ્લોક વાપરવો જોઈએ જેથી જો બીજું સર્વર ડાઉન હોય તો તમારું સર્વર ક્રેશ ના થાય. 
try {
    const res = await axios.post('/api/save', { name: "Palak" });
} catch (err) {
    if (err.response) {
        // સર્વર એ જવાબ આપ્યો પણ સ્ટેટસ કોડ 2xx નથી
        console.log(err.response.status);
    } else {
        // રિક્વેસ્ટ જ ના ગઈ
        console.log(err.message);
    }
}

૩. Async/await in APIs:

બધા જ API કોલ્સ અસિંક્રોનસ (Asynchronous) હોય છે, એટલે કે જવાબ આવવામાં વાર લાગી શકે છે. તેથી async/await વાપરવાથી કોડ ચોખ્ખો અને સિંક્રોનસ જેવો લાગે છે.
એનાલોજી: ઓનલાઇન શોપિંગ. - તમે ઓર્ડર આપો છો (API Request). - હવે જ્યાં સુધી ઓર્ડર ઘરે ના આવે (Response), ત્યાં સુધી તમે બીજા કામ કરી શકો છો (Asynchronous). - પણ જ્યારે ઓર્ડર આવે (Await), ત્યારે જ તમે પેકેટ ખોલીને ચેક કરી શકો છો. જો પેકેટ ફાટેલું હોય તો તમે ફરિયાદ કરો છો (Error Handling).

16. Advanced Node.js Concepts (2-3 yr exp) ⚙️

જ્યારે તમે પ્રોફેશનલ લેવલ પર કામ કરો, ત્યારે કોડનું સ્ટ્રક્ચર અને સિક્યુરિટી બહુ જ મહત્વની બની જાય છે.

૧. MVC Architecture (માળખું):

MVC એટલે કોડને ૩ ભાગમાં વહેંચવો જેથી મેનેજ કરવામાં સરળતા રહે:
  • Model: ડેટાબેઝ સાથે જોડાયેલો ભાગ (Schema).
  • View: જે યુઝરને દેખાય છે (Frontend/UI).
  • Controller: મેઈન લોજિક જે મોડેલ અને વ્યૂ ને જોડે છે.

૨. Folder Structure:

/src
  /controllers  -> લોજિક ફાઈલો
  /models       -> સ્કીમા ફાઈલો
  /routes       -> URL રાઉટ્સ
  /middlewares  -> સિક્યુરિટી ફાઈલો
  /config       -> ડેટાબેઝ કનેક્શન
  server.js     -> મેઈન ફાઈલ

૩. Environment Variables (.env):

સિક્રેટ ડેટા (જેમ કે ડેટાબેઝ પાસવર્ડ કે API Key) ને કોડમાં સીધો લખવાને બદલે .env ફાઈલમાં રાખવો જોઈએ. 
// .env ફાઈલ
PORT=3000
DB_URL=mongodb://localhost:27017

// server.js માં વાપરવા માટે
require('dotenv').config();
console.log(process.env.PORT);

૪. CORS (Cross-Origin Resource Sharing):

જ્યારે તમારું રિએક્ટ (React) બીજા પોર્ટ પર હોય અને નોડ (Node) બીજા પોર્ટ પર હોય, ત્યારે બ્રાઉઝર સિક્યુરિટી માટે રિક્વેસ્ટ રોકી દે છે. તેને મંજૂરી આપવા માટે CORS મિડલવેર વપરાય છે.

૫. Rate Limiting:

એક જ યુઝર વારંવાર રિક્વેસ્ટ મોકલીને સર્વર ડાઉન ના કરી દે, તે માટે રિક્વેસ્ટની સંખ્યા લિમિટ કરવી (દા.ત. ૧ મિનિટમાં ફક્ત ૧૦ રિક્વેસ્ટ).
એનાલોજી: રેસ્ટોરન્ટ. - MVC: રસોડું (Model), મેનુ (View), અને વેઈટર (Controller). - .env: તિજોરીની ચાવી (ગમે ત્યાં ના રખાય, સિક્રેટ જગ્યાએ રખાય). - CORS: હોટલના નિયમો કે બહારનું જમવાનું અંદર નહીં લાવવાનું, જો લાવો તો પરમિશન લેવી પડે. - Rate Limiting: બુફે (Buffet) માં લિમિટ હોવી કે તમે એકસાથે બધું જમવાનું ના લઈ શકો, વારફરતી લેવું પડે.

17. Bonus Topics (Extra Knowledge) 🧠

આ ટોપિક્સ તમારી એપ્લિકેશનને વધુ પાવરફુલ અને રિયલ-ટાઇમ બનાવવામાં મદદ કરે છે.

૧. File Upload (Multer):

જ્યારે યુઝર પ્રોફાઇલ પિક્ચર કે કોઈ ડોક્યુમેન્ટ અપલોડ કરે, ત્યારે તેને હેન્ડલ કરવા માટે Multer મિડલવેર વપરાય છે. 
const multer = require('multer');
const upload = multer({ dest: 'uploads/' });

app.post('/profile', upload.single('avatar'), (req, res) => {
    // ફાઇલ req.file માં મળી જશે
    res.send("ફાઇલ અપલોડ થઈ ગઈ! ✅");
});

૨. WebSockets (Socket.io - Basic Idea):

સામાન્ય API માં ક્લાયન્ટ રિક્વેસ્ટ મોકલે ત્યારે જ સર્વર જવાબ આપે. પણ WebSockets દ્વારા સર્વર અને ક્લાયન્ટ ગમે ત્યારે એકબીજાને મેસેજ મોકલી શકે છે (Real-time).
એનાલોજી: - API: ચિઠ્ઠી લખવી (તમે મોકલો પછી જ સામેથી જવાબ આવે). - WebSockets: ફોન પર વાત કરવી (બંને ગમે ત્યારે બોલી શકે).

૩. Caching (Basic Intro):

વારંવાર વપરાતો ડેટા (દા.ત. પ્રોડક્ટ લિસ્ટ) ને મેમરીમાં સાચવી લેવો જેથી દરેક વખતે ડેટાબેઝ સુધી ના જવું પડે. આનાથી એપ બહુ ફાસ્ટ ચાલે છે. (દા.ત. Redis).
એનાલોજી: રસોડામાં મસાલાના ડબ્બા. તમારે જ્યારે મરચું જોઈએ, ત્યારે તમે સીધું ડબ્બામાંથી લો છો (Cache), તમારે દર વખતે કરિયાણાની દુકાને (Database) જવું પડતું નથી!
