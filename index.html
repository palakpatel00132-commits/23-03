JavaScript Quick Revision 🚀

બધા મહત્વના ટોપિક્સ એકદમ સરળ ગુજરાતીમાં!

1. Var vs Let vs Const �

જાવાસ્ક્રિપ્ટમાં વેરીએબલ ડિકલેર કરવાની ત્રણ રીત છે. આજના સમયમાં 'var' નો ઉપયોગ ટાળવો જોઈએ.
ફીચર var let const
સ્કોપ (Scope) ફંક્શન સ્કોપ બ્લોક સ્કોપ {} બ્લોક સ્કોપ {}
ફરીથી ડિકલેર હા ના ના
વેલ્યુ બદલી શકાય? હા હા ના
રીયલ લાઈફ: - var: જૂનું ઘર જેના દરવાજા ખુલ્લા રહે છે (ગમે ત્યાંથી એક્સેસ). - let: નવું ઘર જેમાં રૂમ પ્રમાણે પ્રાઈવસી છે. - const: એકવાર તાળું મારી દીધું એટલે અંદરનો સામાન ના બદલાય!

2. Closures (ક્લોઝર) 🔒

જ્યારે એક ફંક્શન બીજા ફંક્શનની અંદર હોય, ત્યારે અંદરનું ફંક્શન તેના બહારના ફંક્શનના વેરીએબલ્સને 'યાદ' રાખે છે. 
function outer() {
    let count = 0;
    return function inner() {
        count++;
        return count;
    };
}
const counter = outer();
console.log(counter()); // 1

3. Async Operations (અસિંક્રોનસ કામો) ⏳

જાવાસ્ક્રિપ્ટમાં જ્યારે કોઈ કામમાં વાર લાગે (જેમ કે API માંથી ડેટા લાવવો), ત્યારે તેને હેન્ડલ કરવાની ૩ મુખ્ય રીતો છે:

૧. Callbacks (જૂની રીત):

એક ફંક્શન બીજા ફંક્શનને આર્ગ્યુમેન્ટ તરીકે પાસ કરવામાં આવે છે. આમાં 'Callback Hell' થવાની શક્યતા રહે છે. 
function fetchData(callback) {
    setTimeout(() => {
        callback("ડેટા મળી ગયો! 📦");
    }, 1000);
}

fetchData((result) => {
    console.log(result);
});

૨. Promises (.then / .catch):

Callback Hell થી બચવા માટે ES6 માં પ્રોમિસ આવ્યું. આમાં .then() થી સક્સેસ અને .catch() થી એરર હેન્ડલ થાય છે. 
const myPromise = new Promise((resolve, reject) => {
    setTimeout(() => resolve("પ્રોમિસ પૂરું થયું! ✅"), 1000);
});

myPromise
    .then(res => console.log(res))
    .catch(err => console.log(err));

૩. Async / Await (આધુનિક રીત):

આ પ્રોમિસ પર જ આધારિત છે, પણ કોડ વાંચવામાં 'Synchronous' જેવો લાગે છે. સૌથી વધુ આ જ વપરાય છે. 
async function showData() {
    try {
        const res = await myPromise; // રાહ જોશે
        console.log(res);
    } catch (err) {
        console.log(err);
    }
}

4. Array Methods (એરે મેથડ્સ) 🛠️

જાવાસ્ક્રિપ્ટમાં એરે સાથે કામ કરવા માટે ઘણી બધી મેથડ્સ છે. અહીં સૌથી વધુ વપરાતી મેથડ્સની યાદી છે:

૧. iteration (લૂપિંગ) મેથડ્સ:

  • forEach: એરેની દરેક આઈટમ માટે ફંક્શન ચલાવે છે.
  • map: દરેક આઈટમ બદલીને નવો એરે આપે છે.
  • filter: શરત મુજબ આઈટમ્સ ફિલ્ટર કરીને નવો એરે આપે છે.
  • reduce: બધી આઈટમ્સ ભેગી કરીને એક સિંગલ વેલ્યુ આપે છે.
const nums = [1, 2, 3, 4];

nums.forEach(n => console.log(n)); // ફક્ત પ્રિન્ટ કરશે
const doubled = nums.map(n => n * 2); // [2, 4, 6, 8]
const evens = nums.filter(n => n % 2 === 0); // [2, 4]
const total = nums.reduce((sum, n) => sum + n, 0); // 10

૨. શોધવા (Searching) માટેની મેથડ્સ:

  • find: શરત સંતોષતી પહેલી આઈટમ આપે છે.
  • findIndex: પહેલી આઈટમનો ઇન્ડેક્સ આપે છે.
  • includes: એરેમાં કોઈ વેલ્યુ છે કે નહીં તે (True/False) કહે છે.
const users = [{id: 1, name: "A"}, {id: 2, name: "B"}];

const user = users.find(u => u.id === 2); // {id: 2, name: "B"}
const index = users.findIndex(u => u.id === 2); // 1
const hasThree = [1, 2, 3].includes(3); // true

૩. મોડિફાય (Modify) કરવા માટેની મેથડ્સ:

  • push / pop: છેલ્લે આઈટમ ઉમેરવા / કાઢવા.
  • shift / unshift: શરૂઆતમાં આઈટમ કાઢવા / ઉમેરવા.
  • splice: વચ્ચેથી આઈટમ ઉમેરવા કે કાઢવા.
  • slice: એરેનો અમુક ભાગ કાપીને નવો એરે બનાવવા.
let fruits = ["🍎", "🍌"];

fruits.push("🍇"); // ["🍎", "🍌", "🍇"]
fruits.pop(); // ["🍎", "🍌"]
fruits.shift(); // ["🍌"]
fruits.unshift("🍍"); // ["🍍", "🍌"]

const subArray = [1, 2, 3, 4].slice(1, 3); // [2, 3]

૪. અન્ય મેથડ્સ:

  • sort: એરેને ક્રમમાં ગોઠવવા.
  • some: જો કોઈપણ એક આઈટમ શરત સંતોષે તો True.
  • every: જો બધી જ આઈટમ શરત સંતોષે તો True.
const marks = [40, 50, 30, 80];

const isAnyPass = marks.some(m => m >= 35); // true
const isAllPass = marks.every(m => m >= 35); // false (કારણ કે 30 છે)
const sorted = [10, 5, 80].sort((a, b) => a - b); // [5, 10, 80]

5. Hoisting (હોઈસ્ટિંગ) 🏗️

જાવાસ્ક્રિપ્ટમાં વેરીએબલ કે ફંક્શનને ડિકલેર (Declare) કર્યા પહેલા જ વાપરી શકાય છે, કારણ કે JS એન્જિન તેને કોડમાં સૌથી ઉપર લઈ જાય છે.
રીયલ લાઈફ ઉદાહરણ: પરીક્ષામાં પેપર લખતા પહેલા જ ખબર હોય કે કયા સવાલો પૂછાવાના છે! JS કોડ ચલાવતા પહેલા બધા ડિકલેરેશન વાંચી લે છે.

પ્રેક્ટિસ કોડ:

console.log(a); // undefined (ભૂલ નહીં આવે!)
var a = 10;

sayHi(); // "Hi!" (ફંક્શન પહેલા જ ચાલી જશે)
function sayHi() {
    console.log("Hi!");
}

6. 'this' Keyword 🙋‍♂️

'this' કીવર્ડ હંમેશા એ ઓબ્જેક્ટને પોઈન્ટ કરે છે જે ફંક્શનને અત્યારે ચલાવી રહ્યો છે. તેની વેલ્યુ ક્યાંથી કોલ થાય છે તેના પર બદલાય છે.
રીયલ લાઈફ ઉદાહરણ: "હું" શબ્દ. જો હું બોલું તો "હું" એટલે AI, જો તમે બોલો તો "હું" એટલે યુઝર. કોણ બોલે છે તેના પર "હું" નો અર્થ બદલાય!

પ્રેક્ટિસ કોડ:

const person = {
    name: "રાહુલ",
    greet: function() {
        console.log("મારુ નામ " + this.name + " છે.");
    }
};

person.greet(); // 'this' એટલે person ઓબ્જેક્ટ

7. Arrow Functions 🏹

Arrow functions એ ફંક્શન લખવાની ટૂંકી અને આધુનિક રીત છે. તેમાં 'this' કીવર્ડનું બિહેવિયર અલગ હોય છે (તે બહારના સ્કોપમાંથી 'this' લે છે).

પ્રેક્ટિસ કોડ:

// જૂની રીત
function add(a, b) {
    return a + b;
}

// નવી રીત (Arrow)
const addArrow = (a, b) => a + b;

const sayHello = name => console.log("Hello " + name);

8. Destructuring (ડીસ્ટ્રક્ચરિંગ) ✂️

એરે કે ઓબ્જેક્ટ માંથી વેલ્યુ કાઢીને સીધી વેરીએબલમાં મૂકવાની રીત. 
const user = { name: "Palak", age: 25 };
// જૂની રીત: const name = user.name;
// નવી રીત:
const { name, age } = user;
console.log(name); // Palak

9. Spread & Rest Operator (...) 🌟

- Spread: એરે કે ઓબ્જેક્ટને ખોલીને કોપી કરવા માટે. - Rest: બધી વધારાની આઈટમ્સને ભેગી કરીને એક એરેમાં મૂકવા માટે. 
// Spread: કોપી કરવી
const arr1 = [1, 2];
const arr2 = [...arr1, 3, 4]; // [1, 2, 3, 4]

// Rest: ભેગું કરવું
function sum(...numbers) {
    return numbers.reduce((a, b) => a + b, 0);
}

10. Call, Apply, Bind (કોન્ટેક્સ્ટ બદલવા માટે) 🔗

જ્યારે તમારે 'this' કીવર્ડને કોઈ ચોક્કસ ઓબ્જેક્ટ સાથે જોડવો હોય ત્યારે આ મેથડ્સ વપરાય છે.
  • call: ફંક્શનને તરત જ રન કરે છે (આર્ગ્યુમેન્ટ્સ કોમાથી અપાય છે).
  • apply: ફંક્શનને તરત જ રન કરે છે (આર્ગ્યુમેન્ટ્સ એરે માં અપાય છે).
  • bind: નવું ફંક્શન બનાવીને આપે છે (જેને તમે પછીથી ગમે ત્યારે રન કરી શકો).
const user1 = { name: "રાહુલ" };
function greet(city, state) {
    console.log(`Hello, I am ${this.name} from ${city}, ${state}`);
}

greet.call(user1, "Surat", "Gujarat"); // તરત રન થશે
greet.apply(user1, ["Surat", "Gujarat"]); // એરે સાથે રન થશે
const boundFunc = greet.bind(user1, "Surat", "Gujarat"); // નવું ફંક્શન આપશે
boundFunc(); // પછી રન કરી શકાય

11. ES6 Classes (ક્લાસીસ) 🏛️

જાવાસ્ક્રિપ્ટમાં ઓબ્જેક્ટ ઓરિએન્ટેડ પ્રોગ્રામિંગ (OOP) કરવા માટેની આધુનિક રીત. 
class Animal {
    constructor(name) {
        this.name = name;
    }
    speak() {
        console.log(`${this.name} અવાજ કરે છે.`);
    }
}

class Dog extends Animal { // Inheritance (વારસો)
    bark() {
        console.log(`${this.name} ભસે છે! 🐶`);
    }
}

const myDog = new Dog("Tommy");
myDog.speak();
myDog.bark();

12. Modern JS: Optional Chaining & Nullish Coalescing 💎

આ નવી મેથડ્સ કોડને ક્રેશ થતા બચાવે છે:
  • Optional Chaining (?.): જો ઓબ્જેક્ટમાં કોઈ પ્રોપર્ટી ના હોય, તો એરર આપવાને બદલે 'undefined' આપશે.
  • Nullish Coalescing (??): જો વેલ્યુ null કે undefined હોય, તો જ ડિફોલ્ટ વેલ્યુ લેશે.
const user = { profile: { name: "Palak" } };

// Optional Chaining
console.log(user?.address?.city); // undefined (એરર નહીં આવે!)

// Nullish Coalescing
const score = 0;
const finalScore = score ?? 10; // 0 (કારણ કે 0 null નથી)
const name = null ?? "Guest"; // Guest
