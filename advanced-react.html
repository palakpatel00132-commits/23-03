1. useMemo (પરફોર્મન્સ ઓપ્ટિમાઈઝેશન) 🧠

જ્યારે તમારી પાસે કોઈ ભારે કેલ્ક્યુલેશન (Expensive calculation) હોય, અને તમે નથી ઈચ્છતા કે તે દરેક વખતે ફરીથી ગણાય, ત્યારે useMemo વપરાય છે. તે ગણેલી વેલ્યુને 'યાદ' રાખી લે છે.
રીયલ લાઈફ: ગણિતના અઘરા દાખલાનો જવાબ રફ પેજ પર લખી લેવો. હવે જ્યારે પણ એ જ સવાલ આવે, તો તમારે ગણવો ના પડે, સીધો જવાબ જોઈ શકો!

પ્રેક્ટિસ કોડ:

const result = useMemo(() => {
    return expensiveFunction(data); // ફક્ત ત્યારે જ ચાલશે જ્યારે data બદલાશે
}, [data]);

2. useCallback (ફંક્શન મેમોઈઝેશન) 📞

જ્યારે કમ્પોનન્ટ ફરીથી રેન્ડર થાય ત્યારે જૂના ફંક્શનને ફરીથી બનતા અટકાવવા માટે useCallback વપરાય છે. આનાથી ચાઈલ્ડ કમ્પોનન્ટને વગર કારણે રેન્ડર થતા અટકાવી શકાય છે.
રીયલ લાઈફ: ફોન નંબર સેવ કરવો. જો તમારી પાસે નંબર સેવ છે, તો તમારે દર વખતે ડાયલપેડમાં આંકડા નથી લખવા પડતા, સીધું કોન્ટેક્ટ લિસ્ટમાંથી કોલ કરી શકો.

પ્રેક્ટિસ કોડ:

const handleData = useCallback(() => {
    console.log("હું ફક્ત ત્યારે જ નવો બનીશ જ્યારે ID બદલાશે!");
}, [id]);

3. useRef (ડાયરેક્ટ DOM એક્સેસ) 📍

જ્યારે તમારે કોઈ HTML એલિમેન્ટને સીધો એક્સેસ કરવો હોય (જેમ કે ઇનપુટ પર ફોકસ કરવું) અથવા કોઈ એવી વેલ્યુ સાચવવી હોય જે બદલાય ત્યારે કમ્પોનન્ટને ફરીથી રેન્ડર ના કરે, ત્યારે useRef વપરાય છે.
રીયલ લાઈફ: લેઝર પોઈન્ટર. તમે આખી વસ્તુને હલાવ્યા વગર સીધી કોઈ એક જગ્યાએ પોઈન્ટ કરી શકો છો.

const inputRef = useRef(null);

const focusInput = () => {
    inputRef.current.focus(); // સીધું ઇનપુટ પર ફોકસ કરશે
};

4. Custom Hooks (કોડ રિયુઝેબિલિટી) 🛠️

જ્યારે કોઈ લોજિક (જેમ કે API કોલ કરવો કે વિન્ડો સાઈઝ માપવી) તમારે ઘણા બધા કમ્પોનન્ટમાં વાપરવું હોય, ત્યારે તમે તમારો પોતાનો Custom Hook બનાવી શકો છો. તેનું નામ હંમેશા 'use' થી શરૂ થાય છે.
રીયલ લાઈફ: ઘરની બહાર બનાવેલી પ્લગ-સોકેટ. તમે કોઈ પણ ઇલેક્ટ્રિક વસ્તુ તેમાં ભરાવીને વાપરી શકો છો, તમારે દરેક વસ્તુ માટે નવું વાયરિંગ નથી કરવું પડતું.

કોન્સેપ્ટ એક્ઝામ્પલ:

// useOnlineStatus.js (Custom Hook)
function useOnlineStatus() {
    const [isOnline, setIsOnline] = useState(true);
    // ... લોજિક અહીં આવશે
    return isOnline;
}

// વાપરવા માટે:
const isOnline = useOnlineStatus();

5. Context API (ગ્લોબલ સ્ટેટ) 🌐

જ્યારે તમારે કોઈ ડેટા (જેમ કે 'Theme' કે 'User Login Info') આખા એપ્લિકેશનમાં ગમે ત્યાં વાપરવો હોય, અને તમે નથી ઈચ્છતા કે તેને દરેક કમ્પોનન્ટમાં Props તરીકે પાસ કરવો પડે (Prop Drilling), ત્યારે Context API વપરાય છે.
રીયલ લાઈફ: રેડિયો સ્ટેશન. એકવાર સ્ટેશન પરથી ગીત વાગે એટલે આખા શહેરના લોકો સાંભળી શકે, તમારે દરેકના ઘરે જઈને ગીત સંભળાવવું ના પડે.

પ્રેક્ટિસ કોડ:

const ThemeContext = createContext("light");

// Provider
<ThemeContext.Provider value="dark">
    <App />
</ThemeContext.Provider>

// Consumer
const theme = useContext(ThemeContext);

6. useReducer (કોમ્પ્લેક્સ સ્ટેટ લોજિક) ⚙️

જ્યારે સ્ટેટ મેનેજમેન્ટ થોડું અઘરું હોય (જેમ કે Shopping Cart), ત્યારે useState કરતા useReducer વાપરવું સારું છે. તે Redux જેવું જ કામ કરે છે. 
const initialState = { count: 0 };

function reducer(state, action) {
    switch (action.type) {
        case 'increment': return { count: state.count + 1 };
        default: return state;
    }
}

const [state, dispatch] = useReducer(reducer, initialState);
// dispatch({ type: 'increment' });

7. Redux Toolkit (State Management) 🧠

Redux એ આખી એપ્લિકેશનનો સ્ટેટ એક જ જગ્યાએ (Store) મેનેજ કરવા માટે વપરાય છે. Redux Toolkit (RTK) એ Redux લખવાની આધુનિક અને સરળ રીત છે.

Redux ના પાયાના શબ્દો:

  • Store: એક 'સેન્ટ્રલ ડેટાબેઝ' જ્યાં આખી એપનો સ્ટેટ સાચવવામાં આવે છે.
  • Action: એક ઓબ્જેક્ટ જે કહે છે કે "શું કરવું છે?" (દા.ત. 'ADD_TO_CART').
  • Reducer: એક ફંક્શન જે એક્શન મુજબ સ્ટોરને અપડેટ કરે છે.
  • Dispatch: એક્શનને રિડ્યુસર સુધી મોકલવાની રીત.
Redux Flow:
Component ➔ dispatch(action) ➔ Reducer ➔ Store Update ➔ UI Update

Redux Toolkit Features:

  • createSlice: આમાં એક્શન અને રિડ્યુસર બંને એકસાથે લખી શકાય છે.
  • createAsyncThunk: API કોલ (Async કામો) કરવા માટે વપરાય છે.
રીયલ લાઈફ: બેંક. - Store: બેંકનું તિજોરી (જ્યાં બધા પૈસા છે). - Action: તમારી વિથડ્રોઅલ સ્લિપ (તમારે શું કરવું છે). - Dispatch: કેશિયરને સ્લિપ આપવી. - Reducer: કેશિયર જે તમારી સ્લિપ જોઈને કમ્પ્યુટરમાં બેલેન્સ અપડેટ કરે.

પ્રેક્ટિસ કોડ (createSlice):

import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
    name: 'counter',
    initialState: { value: 0 },
    reducers: {
        increment: (state) => {
            state.value += 1; // RTK માં સીધું સ્ટેટ બદલી શકાય!
        }
    }
});

export const { increment } = counterSlice.actions;
export default counterSlice.reducer;

પ્રેક્ટિસ કોડ (createAsyncThunk):

import { createAsyncThunk } from '@reduxjs/toolkit';

export const fetchUsers = createAsyncThunk(
    'users/fetch',
    async () => {
        const res = await fetch('https://api.example.com/users');
        return res.json();
    }
);

8. API & Full Stack Basics 🌐

જ્યારે આપણો ફ્રન્ટએન્ડ (React) સર્વર (Backend) સાથે વાત કરે છે, ત્યારે તે REST API નો ઉપયોગ કરે છે.

૧. REST API અને HTTP Methods:

  • GET: ડેટા મેળવવા માટે (દા.ત. પ્રોડક્ટ્સ લિસ્ટ જોવું).
  • POST: નવો ડેટા ઉમેરવા માટે (દા.ત. નવું એકાઉન્ટ બનાવવું).
  • PUT: જૂનો ડેટા બદલવા માટે (દા.ત. પ્રોફાઈલ અપડેટ કરવી).
  • DELETE: ડેટા કાઢી નાખવા માટે.

૨. Fetch vs Axios:

- Fetch: બ્રાઉઝરમાં પહેલેથી આવે છે (In-built).
- Axios: એક્સ્ટ્રા લાઈબ્રેરી છે, જે વાપરવામાં સરળ છે અને ઘણા ફીચર્સ આપે છે. 
// Axios Example
import axios from 'axios';

const getData = async () => {
    const res = await axios.get('https://api.example.com/data');
    console.log(res.data);
};

૩. Basic Auth Flow (JWT Concept):

જ્યારે તમે લોગિન કરો, ત્યારે સર્વર તમને એક **"JWT Token"** (એક સિક્રેટ કોડ) આપે છે. આ ટોકન તમે બ્રાઉઝરમાં સાચવી લો છો (Local Storage માં). હવે જ્યારે પણ તમે કોઈ સિક્રેટ ડેટા માંગો, ત્યારે આ ટોકન બતાવવો પડે.
રીયલ લાઈફ: મુવીની ટિકિટ. એકવાર ટિકિટ ખરીદી લો (Login), પછી તમારે વારંવાર પૈસા નથી આપવા પડતા, ફક્ત ટિકિટ (JWT Token) બતાવીને તમે મુવી જોઈ શકો છો.
