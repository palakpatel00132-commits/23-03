DevOps & Production ♾️

Step 7: DevOps અને પ્રોડક્શન ડિપ્લોયમેન્ટ ગુજરાતીમાં!

1. DevOps શું છે? ♾️

DevOps એટલે Development (Dev) અને Operations (Ops) નું મિશ્રણ. તે કોઈ ટૂલ નથી, પણ એક 'સંસ્કૃતિ' (Culture) છે જે સોફ્ટવેર બનાવવાની અને તેને સર્વર પર ચલાવવાની પ્રોસેસને ઝડપી બનાવે છે.
રીયલ લાઈફ: એક એવી હોટલ જ્યાં રસોઈયા (Developers) અને વેઈટર (Operations) એકબીજા સાથે સતત વાત કરે છે, જેથી ઓર્ડર લેવામાં અને પીરસવામાં જરાય વાર ના લાગે!

મુખ્ય ફાયદાઓ:

  • ઝડપી ડિપ્લોયમેન્ટ (Fast Delivery).
  • ઓછી ભૂલો (Fewer Errors).
  • ઓટોમેશન (Automation).
  • સારી સિક્યુરિટી (Security).

2. CI/CD Pipeline 🚀

CI/CD એ DevOps નો પાયો છે. તે કોડ લખવાથી લઈને સર્વર પર મૂકવા સુધીની બધી જ પ્રોસેસને ઓટોમેટ કરે છે.
  • CI (Continuous Integration): જ્યારે પણ તમે કોડ GitHub પર પુશ કરો, ત્યારે તે ઓટોમેટિક ટેસ્ટ થાય અને બિલ્ડ બને.
  • CD (Continuous Deployment): ટેસ્ટ પાસ થયા પછી કોડ ઓટોમેટિક પ્રોડક્શન સર્વર પર જતો રહે.
રીયલ લાઈફ: કાર બનાવવાની ફેક્ટરી. એક બાજુથી કાચો માલ (Code) નાખો, વચ્ચે મશીનો (CI) ચેક કરે અને બીજી બાજુથી તૈયાર કાર (App) બહાર નીકળે!

ટૂલ્સ:

GitHub Actions, Jenkins, GitLab CI, CircleCI.

3. Docker (Containerization) 🐳

Docker તમારા આખા પ્રોગ્રામને (કોડ, લાઈબ્રેરી, સેટિંગ્સ) એક નાના 'ડબ્બા' (Container) માં પેક કરી દે છે. આ ડબ્બો દુનિયાના કોઈ પણ કમ્પ્યુટર પર એકસરખી રીતે ચાલશે.
રીયલ લાઈફ: શિપિંગ કન્ટેનર. તેમાં તમે ફ્રીઝ ભરો કે ટીવી, તે કોઈ પણ જહાજ પર ચઢાવી શકાય અને ક્યાંય પણ મોકલી શકાય. સામાન અંદર સુરક્ષિત રહે છે! 
# સાદું Dockerfile
FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["node", "server.js"]

Docker Compose: એકસાથે ઘણા બધા કન્ટેનર (દા.ત. Node + MongoDB) ચલાવવા માટે વપરાય છે.

4. Kubernetes (K8s) ☸️

જ્યારે તમારી પાસે સેંકડો Docker કન્ટેનર હોય, ત્યારે તેમને મેનેજ કરવા માટે Kubernetes વપરાય છે. તે ચેક કરે છે કે કયું કન્ટેનર ડાઉન છે અને તેને ઓટોમેટિક ફરીથી ચાલુ કરે છે.
રીયલ લાઈફ: જહાજનો કેપ્ટન. તે નક્કી કરે છે કે કયું કન્ટેનર ક્યાં મુકાશે અને જો કોઈ કન્ટેનર પડી જાય તો તેને તરત જ સરખું કરાવે છે.

મુખ્ય ફીચર્સ:

  • Auto-scaling: જો ટ્રાફિક વધે તો ઓટોમેટિક નવા સર્વર ઉમેરે.
  • Self-healing: જો કોઈ કન્ટેનર ક્રેશ થાય તો તેને નવું બનાવી દે.
  • Load Balancing: ટ્રાફિકને બધા સર્વર વચ્ચે સરખે ભાગે વહેંચે.

5. Production Deployment Methods 🏗️

જ્યારે તમે એપનું નવું વર્ઝન (Version) લાવો, ત્યારે યુઝરને કોઈ તકલીફ ના પડે તે માટે અલગ-અલગ રીતો વપરાય છે:

૧. Rolling Deployment (ધીરે-ધીરે):

એક પછી એક જૂના સર્વર બંધ કરીને નવા વર્ઝનવાળા સર્વર ચાલુ કરવા. આમાં એપ ક્યારેય બંધ (Downtime) નથી થતી.

૨. Blue-Green Deployment (બે સર્વર):

બે આબેહૂબ સર્વર સેટઅપ કરવા. 'Blue' માં જૂનું વર્ઝન ચાલે અને 'Green' માં નવું. એકવાર નવું ચેક થઈ જાય એટલે બધો ટ્રાફિક 'Green' પર મોકલી દેવો.

૩. Canary Deployment (ટેસ્ટિંગ):

નવું વર્ઝન ફક્ત ૫% યુઝર્સને જ બતાવવું. જો બધું બરાબર લાગે, તો જ બાકીના ૯૫% ને બતાવવું.

રીયલ લાઈફ: - Rolling: ટ્રેનના ડબ્બા એક-એક કરીને બદલવા. - Blue-Green: નવો પુલ બાજુમાં જ તૈયાર રાખવો, જૂનો બંધ કરીને સીધો નવો ચાલુ કરી દેવો. - Canary: લગ્નમાં જમવાનું બનાવતી વખતે પહેલા થોડા લોકોને ચખાડવું, જો મીઠું બરાબર હોય તો જ બધાને પીરસવું!

6. Monitoring & Logging 📊

એપ પ્રોડક્શન પર ચાલતી હોય ત્યારે તેમાં કોઈ ભૂલ આવે તો તરત ખબર પડવી જોઈએ. તેને 'Observability' પણ કહેવાય છે.
  • Logging: એપમાં શું થઈ રહ્યું છે તેની હિસ્ટ્રી સાચવવી (દા.ત. ELK Stack, Winston).
  • Monitoring: સર્વરની હેલ્થ (RAM, CPU) ચેક કરવી (દા.ત. Prometheus, Grafana).
  • Alerting: જો સર્વર ડાઉન થાય તો ઈમેલ કે મેસેજ મોકલવો.
રીયલ લાઈફ: હોસ્પિટલમાં ICU પેશન્ટનું મોનિટર. તે સતત હૃદયના ધબકારા બતાવે અને જો કંઈક લોચો થાય તો સીટી વાગે!

7. Cloud Providers ☁️

તમારે પોતાનું મોટું સર્વર ખરીદવાની જરૂર નથી, તમે ભાડે લઈ શકો છો. આ સર્વિસ આપતી કંપનીઓને Cloud Providers કહેવાય છે.
Provider માલિક લોકપ્રિયતા
AWS (Amazon Web Services) Amazon સૌથી વધુ વપરાય છે.
Azure Microsoft મોટી કંપનીઓ (Enterprise) માટે બેસ્ટ.
GCP (Google Cloud) Google ડેટા અને AI માટે બેસ્ટ.

DigitalOcean / Heroku / Vercel: નાના પ્રોજેક્ટ્સ માટે સસ્તા અને સરળ ઓપ્શન.

8. Infrastructure as Code (IaC) 📜

સર્વર, ડેટાબેઝ કે નેટવર્કને માઉસથી ક્લિક કરીને સેટ કરવાને બદલે 'કોડ' લખીને સેટ કરવાની રીતને IaC કહેવાય છે.
રીયલ લાઈફ: ઓનલાઇન જમવાનું ઓર્ડર કરવું. તમારે રસોડામાં જવાની જરૂર નથી, ફક્ત મોબાઈલ પર ઓર્ડર (Code) આપો એટલે જમવાનું તૈયાર થઈને આવી જાય!

ટૂલ્સ: Terraform, Ansible, CloudFormation.
